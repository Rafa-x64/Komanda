Aquí tienes la guía paso a paso para construirlo de manera impecable:

Paso 1: Preparar la estructura de archivos
Antes de tirar código, organiza tu espacio de trabajo. Dentro de Komanda-web/src/modules/, asegúrate de tener la siguiente estructura de carpetas (créalas si no existen):

text
src/modules/pos/
├── composables/
│   └── useCart.ts          # Motor lógico reactivo (Carrito)
├── components/
│   ├── ProductCard.vue     # Componente tonto: Tarjeta de producto
│   ├── PosCategoryFilter.vue # Componente tonto: Botones de filtro
│   └── CartItemRow.vue     # Componente tonto: Fila del ticket
├── views/
│   └── POSView.vue         # Componente listo: Orquestador principal
└── pos.api.ts              # Capa de servicio para peticiones HTTP
Paso 2: Construir el Motor Lógico (useCart.ts)
Este será el "cerebro" de tu carrito. Recuerda usar el patrón de Singleton exportando un objeto con variables reactivas y métodos para que mantenga el estado incluso si cambias de vista.

Importa ref y computed de vue.
Define el Estado:
Un ref para cartItems (iniciado como un array vacío de un tipo base { product, quantity, notes }).
Un ref para selectedTable (iniciado en null o 0).
Define las Acciones:
addItem(product): Usa .find() para buscar si el ID del producto ya existe en cartItems. Si existe, increméntale el quantity. Si no, hazle un .push() con cantidad 1 y notas vacías.
updateQuantity(productId, amount): Encuentra el ítem y súmale/réstale el valor. Si la cantidad baja a 0, llama a removeItem.
removeItem(productId): Usa .filter() para sacar el ítem del array.
addKitchenNote(productId, note): Encuentra el ítem y actualiza su campo de notas.
clearCart(): Reinicia cartItems a un array vacío y resetea la mesa.
Define los Computed:
subtotal: Usa .reduce() sobre cartItems multiplicando cantidad por precio.
taxes: Multiplica el subtotal por tu tasa de impuesto (ej. 0.16 para 16%).
total: Suma subtotal + taxes.
Exporta todo dentro de una función o directamente como variables globales reactivas.
Paso 3: Configurar la Capa de Red (pos.api.ts)
Mantén la comunicación con el backend completamente aislada.

Crea una función asíncrona sendOrder(orderPayload).
Usa fetch (o axios si finalmente decides usarlo, aunque hasta ahora el proyecto usa fetch) apuntando a /api/v1/sales/orders con método POST.
Manejo de errores: Envuelve tu envío en un bloque try...catch. Recuerda leer el backend: si el Content-Type no es JSON, lanza un error limpio (para evitar explosiones si el backend te devuelve un HTML de error).
Paso 4: Crear los Componentes "Tontos" (UI)
Estos componentes en src/modules/pos/components/ solo reciben datos por props y avisan al padre cuando algo pasa usando emits. No leen el backend ni stores globales.

PosCategoryFilter.vue:
Props: categories (Array), activeCategory (ID).
Emits: @filter-changed.
Diseño: Usa un contenedor flexbox con desbordamiento horizontal (overflow-x-auto) para que en tablets se pueda hacer scroll con el dedo. Usa botones grandes estilo píldora.
ProductCard.vue:
Props: product (Objeto).
Emits: @add-to-cart.
Diseño: Usa las clases de Bootstrap para tarjetas (.card, .shadow-sm). Ponle la clase cursor-pointer (o configúralo en CSS). Muestra imagen, nombre y precio. Haz que el div.card completo responda al click y emita el evento enviando el producto.
CartItemRow.vue:
Props: item (Objeto del carrito).
Emits: @increase, @decrease, @add-note, @remove.
Diseño: Una flex-row compacta. A la izquierda el nombre, en el centro botones de + y - generosos en tamaño para dedos grandes (btn-lg o padding custom), a la derecha el subtotal del ítem. Añade botones pequeños tipo ícono (lápiz para notas, basurero para eliminar).
Paso 5: Ensamblar la Vista Orquestadora (POSView.vue)
Este es el "Componente Listo" en src/modules/pos/views/. Aquí se junta la magia.

Layout General (El Grid 70/30):
En tu template, usa el sistema de grillas de Bootstrap: un <div class="row">.
Para el lado izquierdo (Catálogo): <div class="col-lg-8"> (o 9).
Para el lado derecho (Ticket): <div class="col-lg-4"> (o 3), con un borde izquierdo o una clase que lo fije visualmente (sticky-top si quieres que no haga scroll con los productos, muy útil en POS).
Lógica del Componente:
Importa tu useCart.ts y extrae sus variables y métodos (cartItems, addItem, total, etc.).
Importa tus componentes tontos y ponlos en el template.
Declara variables reactivas (ref) locales para almacenar los productos y categorías (puedes instanciar unos datos falsos "mock" en onMounted mientras el backend está listo).
Declara un computed para filtrar los productos según la categoría seleccionada en el PosCategoryFilter.
Conexiones entre componentes (El Wiring):
Al catálogo: Haz un v-for sobre tus productos filtrados y úsalos en el <ProductCard>. Escucha e intercepta su evento @add-to-cart="addItem".
Al ticket: Haz un v-for en el módulo de la derecha con <CartItemRow>. Escucha @increase="updateQuantity(item.product.id, 1)" (o pasa la acción que corresponda).
Acción Final (Procesar Orden):
Crea un botón enorme y vistoso al pie de tu ticket derecho: "ENVIAR A COCINA". Muestra el total.
Al presionar el botón, ejecuta un método asíncrono procesarOrden().
En este método, extrae todo lo del cartItems, dale forma de objeto (ej: { tableId: selectedTable.value, items: [...] }), y pásaselo a tu pos.api.ts.
Usa notificaciones (Toast de Bootstrap o alertas nativas por ahora) para avisarle al usuario si salió bien (clearCart()) o si hubo un error.
Paso 6: Configurar las Rutas
Finalmente, tienes que hacer que la aplicación sepa que el POS existe.

Abre el archivo src/router/index.ts. En el array routes, añade la nueva ruta para tu vista inteligente:

javascript
{
    path: '/pos',
    name: 'pos',
    component: () => import('../modules/pos/views/POSView.vue')
}
Tips de oro extra
CSS para Touch: Revisa o añade en src/style.css que los botones y elementos cliqueables desactiven la selección nativa de texto con user-select: none; para evitar que el tablet accidentalmente subraye palabras cuando el cajero presione rápido los botones.
Velocidad: Evita recargos y llamadas innecesarias al servidor mientras montas la UI; haz uso de variables mock hardcodeadas hasta que la parte visual "se sienta bien" en tu dedo/click, luego activas el servicio HTTP real.
